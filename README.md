# Movalo - פלטפורמת SaaS לחברות הובלה

מערכת רב-דיירים (multi-tenant) שנותנת לכל חברת הובלות אתר משלה, מסד נתונים
משלה, הצעות מחיר וחשבוניות אוטומטיות, מעקב GPS אחר משלוחים, ניהול לקוחות
ומזכירה חכמה מבוססת AI - הכל במנוי חודשי אחד. "דוד הובלות" הוא עסק לדוגמה
שמריץ על גבי הפלטפורמה (`/demo`), ומדגים איך נראה האתר של מוביל בודד.

## מבנה האתר (routes עיקריים)

- `/` - דף השיווק הראשי של הפלטפורמה (Movalo) למובילים פוטנציאליים, במגוון
  שפות (עברית, אנגלית, צרפתית, ערבית, רוסית).
- `/demo` - אתר לקוחות לדוגמה ("דוד הובלות") - טופס הצעת מחיר, שירותים, ביקורות וכו'.
- `/register`, `/login` - הרשמה/כניסה של מוביל חדש לפלטפורמה.
- `/dashboard` - הדשבורד של כל מוביל (מוגן ב-JWT, מבודד לפי tenantId).
- `/admin` - דשבורד הניהול הישן (מפתח `x-admin-key`), עדיין בשימוש עבור "דוד הובלות".

## תכונות עיקריות

- 🌍 דף שיווק רב-לשוני לגיוס מובילים חדשים לפלטפורמה
- 🏢 ריבוי-דיירים (multi-tenant): כל מוביל מקבל אתר, מסד נתונים והגדרות משלו
- 🚛 טופס הערכת מחיר אוטומטי + הצעות מחיר להדפסה/PDF
- 🧾 חשבוניות/קבלות תואמות לחוק הישראלי (אמצעי תשלום, ת.ז/ח.פ מעל 5,000 ₪, מספרי הקצאה)
- 📊 דשבורד ניהול מתקדם לכל מוביל
- 🤖 מזכירה/עוזר AI לייעול העסק
- 📍 מעקב GPS בזמן אמת אחרי הובלות
- 📱 ממשק מותאם למובייל, RTL מלא
- 🔔 התראות בזמן אמת
- 📈 דוחות וניתוח נתונים

## התקנה

1. התקן את הדרישות המקדימות:

   ```bash
   node -v  # נדרש Node.js 18 ומעלה
   pnpm -v  # נדרש pnpm 8 ומעלה
   ```

2. התקן את התלויות:

   ```bash
   pnpm install
   ```

3. העתק את קובץ הסביבה:

   ```bash
   cp backend/.env.example backend/.env
   ```

   והגדר את המשתנים הנדרשים, כולל `JWT_SECRET` (חובה עבור הרשמה/כניסה של מובילים).

4. הפעל את הפרויקט:

   ```bash
   # פיתוח - גם frontend וגם backend, עם ניקוי אוטומטי של פורטים תפוסים
   pnpm dev

   # בנייה
   pnpm build

   # הפעלה
   pnpm start
   ```

## מבנה הפרויקט

מונו-רפו מבוסס **pnpm workspaces**, עם חבילת טיפוסים משותפת (`shared`) שמהווה
מקור אמת יחיד לחוזה ה-API בין ה-frontend וה-backend (סטטוסים, שלבי מעקב, צורת
בקשת ההערכה, אמצעי תשלום וכו') - כדי למנוע כפילות טיפוסים וסטייה בין השכבות.

```
dudu-move-site-builder/
├── shared/            # טיפוסים משותפים (backend + frontend) - נבנה ראשון
│   └── src/
│       ├── estimateStatus.ts
│       ├── tracking.ts
│       ├── furniture.ts
│       ├── moveEstimate.ts
│       ├── billing.ts        # אמצעי תשלום, טיפוסי חשבונית/הצעת מחיר
│       ├── businessSettings.ts
│       └── apiResponse.ts
├── frontend/          # React + Vite
│   ├── src/
│   │   ├── components/
│   │   ├── pages/
│   │   │   ├── ForMovers.tsx        # דף השיווק הראשי (multi-tenant SaaS)
│   │   │   ├── Index.tsx            # אתר לקוחות לדוגמה ("דוד הובלות", /demo)
│   │   │   ├── Login.tsx / Register.tsx
│   │   │   ├── admin/                # דשבורד ניהול ישן (x-admin-key)
│   │   │   └── dashboard/             # דשבורד טנאנט חדש (JWT, multi-tenant)
│   │   └── services/
│   └── ...
├── backend/           # Express + TypeScript
│   ├── src/
│   │   ├── controllers/
│   │   ├── services/            # InvoiceService, QuoteService, MongoService...
│   │   ├── middleware/           # requireBusinessAuth (JWT), requireAdminKey (ישן)
│   │   ├── database/models/       # Business, User, MoveEstimate, Customer...
│   │   └── routes/
│   └── ...
└── scripts/
    └── free-port.js    # משחרר פורטים תפוסים אוטומטית לפני pnpm dev
```

`pnpm build` (מהשורש) בונה את החבילות לפי סדר התלויות - קודם `shared`, ואז
`backend`/`frontend` שתלויים בה דרך `"shared": "workspace:*"`.

## ריבוי-דיירים (Multi-tenant)

כל מוביל שנרשם דרך `/register` מקבל חשבון עסק (`Business`) עצמאי עם JWT משלו.
כל הנתונים (בקשות הובלה, לקוחות, הגדרות עסק, הערות יומן) מבודדים לפי `tenantId`
בכל שכבות המערכת (מסד נתונים, שירותים, controllers) - ר' `backend/src/lib/tenantFilter.ts`.
"דוד הובלות" ממשיך לפעול במקביל תחת המסלול הישן (`x-admin-key`, ללא tenantId)
כדוגמה חיה ולקוח קיים של המערכת.
